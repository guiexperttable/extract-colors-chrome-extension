Privacy Policy – Extract Colors DevTool

Effective date: 2025-09-06

Purpose Data Collection & Use Permissions Retention & Storage User Controls Security Children Changes Contact

Sole Purpose

The extension's sole purpose is to help designers and developers extract, inspect, and organize colors from web pages they choose, and to provide auxiliary visual tools (e.g., ruler, screen capture preview) that assist in color analysis. All processing happens locally in the user's browser. The extension does not track browsing behavior, does not build user profiles, and does not transmit page content or extracted colors to external servers.

Disclosure:
  • We do not sell or transfer user data to third parties, except as necessary for the extension’s core functionality where permitted by the Chrome Web Store policies.
  • User data is not used or transferred for purposes unrelated to the extension’s sole purpose.
  • User data is not used or transferred to determine creditworthiness or for lending purposes.
  • No remote code execution or remotely hosted code is used.

Data Collection and Use

The extension performs processing locally within the browser to sample colors, display overlays (such as rulers), and generate on-screen previews. The extension does not collect or transmit page content, extracted colors, or browsing history to external servers. No tracking or profiling is performed.

Locally Stored Items

We store the following items locally via chrome.storage to provide a consistent experience:

  • User preferences (e.g., UI options, last used formats)
  • Saved color palettes and recent color history
  • Tool states (e.g., ruler settings)

Data is scoped to the extension and used only to provide and improve color extraction and related tools.

Permissions and Justifications

  • storage – Used to save user preferences, saved palettes, recent color history, and tool states. Enables a consistent experience across sessions and devices (if sync is enabled). No personal data is stored. Data is scoped to this extension and used only to provide and improve the color extraction workflow.
  • unlimitedStorage – Color collections, screenshots/previews, and history can exceed default quota. This prevents users from being forced to delete design assets or settings due to size limits. Data remains local; no external transfer occurs.
  • activeTab – Allows temporary access to the current tab after an explicit user action (e.g., clicking the extension). Required to sample on‑page colors, read computed styles, overlay tools (like the ruler), and generate previews. Not used to access other tabs or run without explicit user intent.
  • scripting – Injects lightweight scripts into the active page to sample pixel colors under the cursor, read computed styles, render overlays (rulers/guides), and support screen capture previews. Injection occurs only after user initiation. Scripts do not exfiltrate content; processing stays local and only for the active page.
  • system.display – Reads display characteristics (resolution, scale factor, color space hints) to render accurate overlays/rulers and size screenshots/previews correctly. Improves color fidelity and measurement precision across monitors. The extension does not collect or transmit device‑identifying display data.
  • browsingData – Used solely to allow the user to clear the extension’s own cached data and temporary artifacts created during color extraction, previews, or screenshots. The extension does not delete unrelated browsing history or data without an explicit user request.

Data Retention and Storage

Data you save (e.g., palettes, preferences) remains stored locally in Chrome’s extension storage. If Chrome Sync is enabled, eligible items may sync across your devices under your Google account, per Chrome’s own settings. We do not operate external servers for storage.

You may clear the extension’s data using built-in controls (see below). When removed, associated local data will be deleted. We do not retain backups.

User Controls and Transparency

  • Explicit activation: Functionality that touches the current tab (e.g., sampling colors, overlays) only runs after a clear user action.
  • Clear data: Use the extension’s “Clear cache/data” option to remove its stored items and temporary artifacts.
  • Chrome controls: You can also remove the extension or clear site/extension data via Chrome’s settings.

Security

We aim to minimize data collection and keep processing local. We do not transmit page content or extracted colors to external services. The extension does not execute remote code.

Children

This extension is a developer/design utility and is not directed to children. It does not knowingly collect personal information from children.

Changes to This Policy

We may update this policy to reflect changes in functionality or legal requirements. Material changes will be reflected by updating the effective date and publishing the updated policy within this page.

Contact
